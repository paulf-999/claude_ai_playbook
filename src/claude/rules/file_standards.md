# 🧹 Rules — File standards

These are enforced on every commit:

- ✅ Files must end with a single newline
- ✅ No trailing whitespace
- ✅ No UTF-8 BOM
- ✅ Consistent line endings (no mixed CRLF/LF)
- ⚠️ No large files committed
- ⚠️ No filename case conflicts (safe for case-insensitive filesystems)
- 🔐 No private keys or secrets in committed files
