# Standard ADF page header

All `create_confluence_page` patterns must include this header at the start of every generated page. Read this file and include the nodes below — in order — as the first items in the ADF `content` array.

Also read `~/.claude/skills/create_confluence_page/_common/_common_formatting.md` and apply those formatting rules throughout the entire page body.

---

## 1. Note panel — Claude Code

```json
{
  "type": "panel",
  "attrs": {"panelType": "note"},
  "content": [
    {"type": "paragraph", "content": [
      {"type": "text", "text": "Note: this page was generated by Claude Code."}
    ]}
  ]
}
```

---

## 2. Page Status — status badge

Colour mapping — always one badge, matching the selected status:

| Status | color value |
|---|---|
| Not started | `red` |
| In progress | `yellow` |
| Ready for review | `blue` |
| Complete | `green` |

```json
{
  "type": "paragraph",
  "content": [
    {"type": "text", "text": "Page Status", "marks": [{"type": "strong"}]},
    {"type": "text", "text": ": "},
    {
      "type": "status",
      "attrs": {
        "text": "<STATUS_LABEL>",
        "color": "<COLOR>",
        "style": "bold",
        "localId": "<uuid>"
      }
    }
  ]
}
```

## 3. Header metadata (one paragraph each)

```json
{"type": "paragraph", "content": [{"type": "text", "text": "Creator", "marks": [{"type": "strong"}]}, {"type": "text", "text": ": <name>"}]}
{"type": "paragraph", "content": [{"type": "text", "text": "Version", "marks": [{"type": "strong"}]}, {"type": "text", "text": ": 1.0"}]}
{"type": "paragraph", "content": [{"type": "text", "text": "Date Created", "marks": [{"type": "strong"}]}, {"type": "text", "text": ": <today's date>"}]}
```

---

## 4. Purpose — info panel

Draft the purpose statement as a single, brief, high-level sentence if the user has not provided one.

```json
{
  "type": "panel",
  "attrs": {"panelType": "info"},
  "content": [
    {"type": "paragraph", "content": [
      {"type": "text", "text": "The purpose of this page is to …"}
    ]}
  ]
}
```

---

## 5. Contents heading + TOC macro + rule

```json
{"type": "heading", "attrs": {"level": 1}, "content": [{"type": "text", "text": "📋 Contents"}]}

{
  "type": "extension",
  "attrs": {
    "extensionType": "com.atlassian.confluence.macro.core",
    "extensionKey": "toc",
    "parameters": {
      "macroParams": {
        "minLevel": {"value": "1"},
        "maxLevel": {"value": "2"},
        "exclude": {"value": "Contents"}
      }
    },
    "layout": "default"
  }
}

{"type": "rule"}
```

## ADF body conventions

Use these patterns consistently throughout the body sections:

- **Headings:** `{"type": "heading", "attrs": {"level": 1}, "content": [{"type": "text", "text": "🔍 1. Section Name"}]}`
- **Bullet points:** `bulletList` > `listItem` > `paragraph`
- **Ordered list:** `orderedList` > `listItem` > `paragraph`
- **Tables:** `table` > `tableRow` > `tableHeader` / `tableCell` > `paragraph`
- **Rule dividers:** `{"type": "rule"}` — place after each section
- **Bold text:** `"marks": [{"type": "strong"}]` on text nodes
- **Inline code:** `"marks": [{"type": "code"}]` on text nodes

### Default emoji mapping for section headings

| Section type | Emoji |
|---|---|
| Overview / Summary | 🔍 |
| How it works / Process / Steps | ⚙️ |
| Reference / Checks / Details | 📌 |
| Requirements / Criteria | 📋 |
| Background / Context | 📖 |
| Timeline | 🕐 |
| Actions / Next steps | ✅ |
| Risk / Assessment | ⚠️ |
| Recommendation / Idea | 💡 |
| Decision | 🏛️ |
| Blocked | 🚧 |
